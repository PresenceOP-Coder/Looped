# Security Policy

## Supported Versions

Security fixes are prioritized for the latest code on `main`.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

1. Share details privately with project maintainers.
2. Include reproduction steps, impact, and affected area.
3. If possible, include a minimal proof of concept.

Response target:

- Initial acknowledgement within 72 hours
- Triage and next steps as soon as validated

## Sensitive Data

- Never commit API keys, service account files, tokens, or secrets.
- Use environment variables or local, ignored files for credentials.
